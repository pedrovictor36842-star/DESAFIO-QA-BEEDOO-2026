# DESAFIO-QA-BEEDOO-2026

Objetivo da Aplicação
O objetivo desta aplicação é fornecer uma interface centralizada para o gerenciamento de cursos de uma instituição. O sistema permite que administradores criem, visualizem e mantenham um catálogo organizado, garantindo que todas as informações dos cursos (como nome, descrição, instrutor, datas de início e término e modelo de aprendizado) estejam devidamente registradas e acessíveis aos usuários.

Principais Fluxos Disponíveis

Fluxo de Cadastro de Cursos: Registro de novos cursos no sistema através do preenchimento de formulários.
Fluxo de Listagem de Cursos: Exibição organizada do catálogo para visualização e conferência dos dados.
Fluxo de Manutenção: Funcionalidades de edição e exclusão de registros para a gestão contínua do catálogo.

Pontos Críticos para Teste

Validação de Campos Obrigatórios: Verificar se o sistema impede o processamento quando campos essenciais não são preenchidos e se exibe mensagens de erro claras e precisas.
Validação de Saída de Respostas: Garantir que o sistema retorne as informações corretas e no formato esperado após a interação do usuário.
Tratamento de Erros: Avaliar a resiliência do sistema diante de falhas, especificamente em tentativas de exclusão de registros inexistentes ou em falhas de comunicação com o banco de dados, assegurando que o sistema não apresente comportamento instável.
